# customcode-managed environment

This machine's Claude environment is carried here by **customcode**: `CLAUDE_CONFIG_DIR` points at this directory, which is synced from the user's **claude home repo** (this content) plus customcode's own machinery whenever the user triggers a config sync.

- `CLAUDE.md`, `settings.json`, `skills/` (and `keybindings.json`, `agents/`, `commands/`, `statusline-render` if present) come from the claude home repo configured in customcode's Rucksack. Local edits are overwritten on every sync, and files added under `skills/` are **deleted**. If the user asks to add or change a global skill or setting here, warn them it will not survive — it must be changed in the claude home repo instead.
- `statusline.py`, `kimi.sh`, `zdotdir/`, and the `statusLine`/`tui` keys in `settings.json` are **customcode machinery**, overlaid at every sync regardless of what the repo contains — never edit them here and never ship real ones in the repo.
- Runtime state persists across syncs. `.credentials.json` in this directory is this machine's customcode login — never modify or delete it.
- Claude is deliberately **stateless** here: memory or notes about the current project belong in the project itself — its committed `CLAUDE.md` / `.claude/` — so every developer working on the project shares the same context. If the user asks you to remember something, store it project-local; never write memory into this directory.
- The project's own `.claude/` directory works normally and is not managed by customcode.
- **Git/GitHub goes through `gh` over HTTPS.** Machines have no GitHub ssh keys: use the `gh` CLI for GitHub operations (`gh repo clone`, PRs, issues) and HTTPS remotes for plain git — `gh auth setup-git` wires git's credential helper to gh, so `git clone`/`push` over HTTPS just work. An ssh-form URL (`git@github.com:owner/repo`) still fails with `Permission denied (publickey)` even after setup-git — rewrite it to `https://github.com/owner/repo`, or make that automatic once per Machine with `git config --global url."https://github.com/".insteadOf git@github.com:`. If an HTTPS git command fails with an auth error, run `gh auth login` then `gh auth setup-git`; never switch a remote to ssh to work around it.

## Notebook

customcode has a **Notebook** per project (plus a global/personal one) holding typed items — **todos**, **notes**, and **prompts** — visible in a panel next to the terminals. Inside any customcode terminal you can ask Claude to "add this as a todo to my notebook", "note this down", or "save this prompt" — the `customcode-notebook` skill writes it over the terminal-token API and it appears in the open panel instantly. Items land in the current project's notebook by default; say "global" for the personal notebook that rides into every project. Prompts saved there can be clicked in the UI to insert their text into the focused terminal without submitting. Claude creates items freely on request but only edits or deletes items you explicitly point at.
